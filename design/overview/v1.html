Summary card — responsive (the cramming fix)

The donut sits beside the figures while they're short, and drops below them when a total runs long (large JPY/INR), so a big number never squeezes the chart. The "% of trip spend" line is gone. In the app this is ViewThatFits — it auto-picks whichever layout fits, and only shrinks text inside a layout as a last resort.

Decisions reflected here — Overview is spend-only (settlements excluded everywhere; net-owed lives on Balances). Everything is scoped to one currency via the picker pill (hidden when a trip uses a single currency). Category colours are SageColors.CategoryTone, shared between the category breakdown and the daily stacked bars. "Spend" is shown as both paid (fronted) and share (consumed) so neither sense of "who spent the most" is hidden. Daily bars bucket by expense date.

To pick: one of Directions A / B / C as the base layout (mix-and-match welcome — e.g. B's person table inside A's donut summary). The three states (picker / single-currency / empty) apply to whichever base wins.
