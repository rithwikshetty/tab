Settle up — mid-trip settlement flow

Record partial or full cash payments between people mid-trip. Settlements reduce balances but don't count toward expense totals. Entry point is the three-dots menu on the trip detail screen.

Entry point — three-dots menu

Trigger

"Settle up" lives in the existing three-dots menu alongside "Export to Excel". Tapping it opens the settlement form as a sheet.

How it works

Settle up is a secondary action in the three-dots menu. The form follows the same patterns as expense entry: Cancel/Save nav bar, large amount display, card-based form fields.

Element Where Behaviour
Entry point Three-dots menu "Settle up" menu item. Opens form as a sheet.
From dropdown Form, payment card Defaults to current user. Tap to pick any trip member.
To dropdown Form, payment card Defaults to person the "From" user owes the most. Tap to change.
Amount Form, top Pre-filled with full owed balance. Tappable to edit for partial settlement.
Balance context Form, below card Blue info banner showing current balance and what remains after settlement. Updates dynamically.
Timeline row Expense list Compact row (~half expense height), blue accent, arrow icon. "Alice settled with Bob" + amount. Tappable for detail.
Settlement detail Push page Read-only view with from/to, amount, date, note, recorder. Delete in nav bar.
Key decisions: Settlements use a distinct blue accent (not sage green) to visually separate them from expenses. The balance context banner is the key UX detail — it tells you what you owe, what you're settling, and what will remain. When the user changes "From" or "To", the amount auto-updates to the full balance between that pair. The compact timeline row is intentionally shorter and simpler than expense tiles — settlements are supporting entries, not primary content. If "From" and "To" are set to the same person, Save is disabled.
