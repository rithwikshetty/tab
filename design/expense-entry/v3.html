How it works

One combined page replaces the old separate "Paid by" push page. The main form gets a compact summary row instead of two separate rows + a segmented control.

Element Where Behaviour
Summary row Main form "You · Equal split" — tap to push to Payment & Split page. Updates on return.
Who paid Push page, top Checkbox per member. Single payer = amount shown. Multi-payer = editable fields + reconciliation footer.
Split type pill Push page, section header Dropdown: Equal, Exact (now), Percentage, Shares (future). Changes participant list behaviour.
Split between Push page, bottom Checkbox per member. Equal = auto-calculated. Exact = editable fields + reconciliation footer.
Participant list Main form, below card Read-only preview of current split. Shows amounts but no edit controls.
Date Main form, in card Unchanged — stays as its own row in the card below the summary row.
Key decisions: The push page title is "Payment & Split" — rename to whatever feels right. The participant list on the main form is read-only (view-only context); all editing happens on the push page. The split type dropdown only lives on the push page, not the main form — keeps the main form clean. When the payer is just "You" (90% case), the Who Paid section shows amounts but they're not editable — nothing to edit. The summary row subtitle adapts: "Split between 3 people" (default) vs. "You, Alice, Bob paid · 3-way split" (customised).
